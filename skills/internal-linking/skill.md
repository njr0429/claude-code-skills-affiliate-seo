# Internal Linking Skill

## 概要
SEO効果を最大化する内部リンク戦略を設計し、サイト全体の回遊率とクローラビリティを向上させる

---

## Inputs（入力）

### 必須
- `site_url`: サイトURL（または記事リスト）

### オプション
- `platform`: プラットフォーム（`blogger` / `wordpress`、デフォルト: `blogger`）
- `article_count`: 記事数（デフォルト: 自動カウント）
- `focus_articles`: 重点的に最適化したい記事URL（収益記事など）

---

## Outputs（出力）

### 形式
Markdown形式の内部リンク最適化プラン

### 含まれる内容
1. 現状分析（記事分類、被リンク数、孤立ページ）
2. トピッククラスター設計（ピラー記事とクラスター記事）
3. アクションプラン（優先度付き）
4. リンク追加例（具体的なアンカーテキスト）
5. 期待効果（PV増、CVR向上予測）

---

## このスキルを使うタイミング
- 記事が10本以上たまった後
- 収益記事への導線を強化したいとき
- サイト全体の内部リンク構造を見直すとき
- 孤立ページ（内部リンクゼロ）を発見したとき

---

## 実行手順

### 1. 現状分析

#### サイトマップ確認
全記事を以下で分類:
```markdown
## 記事分類

### 収益記事 (5-10記事)
| 記事 | URL | 現在の被リンク数 | 目標 |
|------|-----|----------------|------|
| 記事A | /url-a/ | 2個 | 5個以上 |
| 記事B | /url-b/ | 1個 | 5個以上 |

### 集客記事 (20-40記事)
| 記事 | URL | 発リンク数 | 収益記事へのリンク |
|------|-----|-----------|------------------|
| 記事C | /url-c/ | 1個 | なし ❌ |
| 記事D | /url-d/ | 3個 | あり ✅ |

### 孤立ページ (内部リンク0)
- 記事E - /url-e/ ❌ 要対応
```

### 2. トピッククラスター構造の設計

#### ピラミッド型リンク構造
```
           TOPページ
               |
      ┌────────┼────────┐
      |        |        |
  カテゴリA カテゴリB カテゴリC
      |        |        |
   ピラー    ピラー    ピラー
   記事      記事      記事
   (収益)    (収益)    (収益)
   ↑ ↑ ↑    ↑ ↑ ↑    ↑ ↑ ↑
   | | |    | | |    | | |
  クラスター クラスター クラスター
  記事群    記事群    記事群
  (集客)    (集客)    (集客)
```

#### トピッククラスター例
```markdown
## クラスター1: スキンケア基礎

### ピラー記事 (収益)
- 「スキンケアおすすめランキングTOP10」

### クラスター記事 (集客)
- 「スキンケアとは？初心者向けガイド」→ ピラーへリンク
- 「肌タイプの見分け方」→ ピラーへリンク
- 「スキンケアの正しい順番」→ ピラーへリンク
- 「年代別スキンケア方法」→ ピラーへリンク
- 「季節別スキンケアのコツ」→ ピラーへリンク

**リンク方向**: クラスター → ピラー (5本の矢印)
```

### 3. 内部リンク戦略

#### リンク設計ルール

**1. 集客記事 → 収益記事 (必須)**
```markdown
- 各集客記事から収益記事へ: 2-3箇所
- アンカーテキスト: 自然で関連性の高い文言
- 配置: 本文中、自然な文脈で
```

**2. 収益記事 → 関連収益記事 (任意)**
```markdown
- 比較・代替案として: 1-2箇所
- 例: 「A商品」記事 → 「B商品」記事
- 「他のおすすめ商品はこちら」
```

**3. 新記事 → 古い記事 (SEO強化)**
```markdown
- 新記事から古い記事へ: 3-5箇所
- 古い記事のSEO評価を底上げ
- 特に順位11-30位の記事を狙う
```

**4. 古い記事 → 新記事 (リライト時)**
```markdown
- リライト時に新記事へリンク追加
- 関連性の高い記事同士を相互リンク
```

#### アンカーテキスト最適化

```markdown
❌ 悪い例:
- 「こちら」
- 「詳細はこちら」
- 「クリック」
- 「この記事」

✅ 良い例:
- 「初心者向けスキンケアの選び方」
- 「敏感肌におすすめの化粧水ランキング」
- 「30代のエイジングケア方法」
- 「[商品名]の口コミ・評判を見る」

ポイント:
- キーワードを含める
- 具体的な内容を示す
- 自然な日本語
- 30文字以内推奨
```

### 4. 内部リンク配置の最適化

#### 記事内の配置箇所

```markdown
## 優先度★★★ (必須)

### 1. 本文中の自然な文脈 (2-3箇所)
「〇〇について詳しく知りたい方は、[XXXの選び方ガイド](url)をご覧ください。」

### 2. 関連記事セクション (記事下部)
```
## あわせて読みたい
- [関連記事1のタイトル](url)
- [関連記事2のタイトル](url)
- [関連記事3のタイトル](url)
```

### 3. カテゴリーページ
各カテゴリーページから全記事へリンク

## 優先度★★☆ (推奨)

### 4. サイドバー
- 人気記事TOP5
- カテゴリー一覧
- 新着記事

### 5. パンくずリスト
```
ホーム > カテゴリー > 記事タイトル
```

### 6. 記事内の補足ボックス
```
💡 関連情報
[XXXについて詳しくはこちら](url)
```
```

### 5. リンクジュース（評価）の流れ最適化

#### PageRank分配の考え方
```
TOPページ (DR最高)
   ↓ リンクジュース
カテゴリーページ
   ↓ リンクジュース
ピラー記事 (収益記事)
   ↑ リンクジュース
クラスター記事 (集客記事)

目的: 収益記事にリンクジュースを集める
```

#### nofollow使用ルール
```markdown
✅ nofollowをつける:
- アフィリエイトリンク (必須)
- 外部サイトへのリンク (信頼できないサイト)
- お問い合わせ、プライバシーポリシー

❌ nofollowをつけない:
- 内部リンク (全て)
- 信頼できる公式サイトへのリンク
```

### 6. 回遊率向上施策

#### 関連記事の自動表示
```markdown
## 記事下の関連記事

### 表示ルール
1. 同じカテゴリーの記事: 3記事
2. 関連キーワードを含む記事: 2記事
3. 人気記事: 1記事

### 並び順
- 優先度1: 収益記事
- 優先度2: 関連性の高い集客記事
- 優先度3: PVの多い記事
```

#### 記事内リンクの工夫
```markdown
### 1. 目次リンク (ページ内)
```
## 目次
1. [セクション1](#section1)
2. [セクション2](#section2)
```
→ 離脱率を下げる

### 2. 「続きを読む前に」リンク
```
📌 この記事を読む前に
まずは[基礎知識](url)を理解しておくとスムーズです。
```

### 3. 「次に読むべき記事」
```
✅ この記事を読んだら次は
[実践編の記事](url)で具体的な手順を学びましょう。
```
```
