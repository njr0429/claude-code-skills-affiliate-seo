# SEO Article Generator Skill

## 概要
検索上位を狙えるSEO最適化された高品質記事を効率的に作成する

---

## Inputs（入力）

### 必須
- `target_keyword`: 対象キーワード
- `article_type`: 記事タイプ（`revenue` = 収益記事 / `traffic` = 集客記事）

### オプション
- `platform`: プラットフォーム（`blogger` / `wordpress`、デフォルト: `blogger`）
- `target_word_count`: 想定文字数（デフォルト: 競合平均+10-15%）
- `affiliate_product`: アフィリエイト商品情報（収益記事の場合）
- `competitor_analysis_result`: 競合分析結果（/competitor-analysisの出力）

---

## Outputs（出力）

### 形式
Markdown形式の完成記事

### 含まれる内容
1. メタ情報（タイトル、description、キーワード）
2. H1タイトル
3. 導入文（PREP法）
4. 目次
5. 本文（H2-H3構成）
6. まとめ
7. 関連記事リンク候補
8. 使用画像リスト（alt属性付き）
9. 品質チェックリスト

---

## このスキルを使うタイミング

### ✅ 使うべきとき
- **新規記事を作成するとき（/competitor-analysis 完了後）**
- 既存記事をリライトするとき
- 収益記事・集客記事の執筆
- コンテンツカレンダーに基づいて記事作成を進めるとき

### ❌ 使わない方がいいとき
- キーワードが決まっていない段階（/keyword-research を先に）
- 競合分析していない段階（/competitor-analysis を先に推奨）
- サイト構造が決まっていない段階（/site-architecture を先に）

### 📍 ワークフロー上の位置
```
/competitor-analysis → このスキル → /internal-linking（記事10本超えたら）
```

## 実行手順

### 1. 記事企画ヒアリング
ユーザーに以下を質問:
- 対象キーワード
- 記事タイプ (収益記事 / 集客記事)
- 想定文字数 (2,000-8,000文字)
- アフィリエイト案件 (収益記事の場合)

### 2. 競合記事分析
上位10記事を分析:
- 見出し構成
- 文字数
- 独自コンテンツ
- 不足している情報

### 3. 記事構成案作成

```markdown
# [タイトル (30-40文字、キーワード含む)]

## メタディスクリプション (120文字以内)
[検索結果に表示される説明文]

## 目次
1. [導入]
2. [H2見出し1] - キーワードの基礎知識
3. [H2見出し2] - 具体的な方法・手順
4. [H2見出し3] - おすすめ商品・サービス ★収益ポイント
5. [H2見出し4] - よくある質問
6. [まとめ]

内部リンク挿入箇所: 3箇所
アフィリエイトリンク: 2-3箇所
画像: 5枚以上
```

### 4. 記事本文生成

#### E-E-A-T を意識した執筆
- **Experience (経験)**:
  - ✅ 実体験がある場合のみ明記する（「実際に使ってみた」など）
  - ✅ 実体験がない場合は比較・調査ベースで書く（「〇〇を比較調査した結果」）
  - ❌ 架空の体験談を作らない（景品表示法違反リスク）
- **Expertise (専門性)**: 専門的な知識を示す、最新情報にアップデート
- **Authoritativeness (権威性)**: 信頼できる情報源を引用（公式サイト、統計データ）
- **Trustworthiness (信頼性)**: 正確で偏りのない情報、出典明記

#### SEO最適化ポイント
```
✅ タイトルタグ: 主要キーワード前方配置、30-40文字
✅ H1タグ: タイトルと同じ
✅ H2-H3: 関連キーワード自然に含める
✅ 最初の100文字: キーワード含める
✅ キーワード配置: タイトル、導入、見出し、まとめに自然に含める
  ❌ 機械的な出現率管理（1-2%など）は避ける
  ✅ 不自然な繰り返しを避ける
✅ 共起語・関連語: テーマ網羅を重視、自然に散りばめる
✅ 文字数: 競合の平均+10%以上
✅ 内部リンク: 3-5箇所
✅ 外部リンク: 公式サイト、統計データなど（出典明記）
✅ 画像: 必要に応じて挿入（alt属性にキーワード含める）
  - 図解が必要な箇所に配置
  - 枚数よりも質と関連性を重視
```

#### 収益記事の場合
```markdown
## おすすめ[商品カテゴリ]TOP3

### 1位: [商品名]
[商品画像]

**おすすめポイント**
- ポイント1
- ポイント2
- ポイント3

**こんな人におすすめ**
- タイプ1
- タイプ2

**口コミ・評判**
> [出典が確認できる口コミのみ引用]
> ※出典: Amazon/楽天レビュー、公式サイト等

**注意**: 架空の口コミを作成しない

[アフィリエイトボタン]

---

### 2位: [商品名]
...
```

### 5. コンバージョン導線設計

記事内のCV導線:
1. **導入部分**: 問題提起 → 解決策の提示
2. **中盤**: 詳細情報 → 比較表・ランキング
3. **終盤**: CTA (Call to Action)

#### Blogger版（シンプルHTML）
```html
<!-- CTAボタン例（Blogger） -->
<p style="text-align:center; padding:20px; background:#f9f9f9; border-radius:8px; margin:30px 0;">
  <strong style="display:block; margin-bottom:10px; color:#333;">今すぐ公式サイトで詳細をチェック！</strong>
  <a href="[アフィリエイトリンク]" target="_blank" rel="nofollow" style="display:inline-block; padding:12px 30px; background:#0066cc; color:#fff; text-decoration:none; border-radius:5px; font-weight:bold;">
    公式サイトはこちら
  </a>
</p>
```

#### WordPress版（カスタムCSS利用可）
```html
<!-- CTAボタン例（WordPress） -->
<div class="cta-box">
  <p class="cta-text">今すぐ公式サイトで詳細をチェック！</p>
  <a href="[アフィリエイトリンク]" class="cta-button" target="_blank" rel="nofollow">
    公式サイトはこちら
  </a>
</div>
```

### 6. 品質チェックリスト

記事完成後、以下を確認:

#### 基本SEO
- [ ] タイトルにキーワードが含まれている
- [ ] メタディスクリプションが120文字以内
- [ ] H2見出しが3つ以上ある
- [ ] 文字数が2,000文字以上
- [ ] キーワードが自然に配置されている（機械的な繰り返しなし）

#### コンテンツ品質
- [ ] 画像・図表が必要な箇所に配置されている（枚数より質重視）
- [ ] 内部リンクが3箇所以上ある
- [ ] 外部リンクが1つ以上ある（出典明記）
- [ ] 誤字脱字がない
- [ ] スマホで読みやすい（短い段落、3-4行）
- [ ] オリジナル性がある（コピペではない）

#### E-E-A-T・法令遵守（重要）
- [ ] 実体験がある場合のみ「使ってみた」等の表現を使用
- [ ] 実体験がない場合は調査・比較ベースで記述
- [ ] 口コミは出典が確認できるもののみ引用（出典明記）
- [ ] 架空の体験談・レビューを作成していない
- [ ] 誇張表現・断定表現を避けている

## 記事タイプ別テンプレート

### 商標キーワード記事
```
タイトル例: 「[商品名]の口コミ・評判は?実際に使った感想を正直レビュー」

構成:
1. [商品名]とは
2. [商品名]の特徴・スペック
3. 実際に使った口コミ・評判
4. メリット・デメリット
5. こんな人におすすめ
6. 最安値で購入する方法
7. よくある質問
8. まとめ
```

### 比較・ランキング記事
```
タイトル例: 「[カテゴリ]おすすめ10選!目的別ランキングと選び方」

構成:
1. [カテゴリ]の選び方
2. おすすめランキングTOP10
3. 目的別おすすめ
4. 比較表
5. よくある質問
6. まとめ
```

### HowTo記事 (集客)
```
タイトル例: 「[〇〇のやり方]初心者でも5分でできる簡単ガイド」

構成:
1. [〇〇]とは
2. 準備するもの
3. 手順1
4. 手順2
5. 手順3
6. よくある失敗と対処法
7. さらに詳しく学ぶなら (内部リンク)
8. まとめ
```
