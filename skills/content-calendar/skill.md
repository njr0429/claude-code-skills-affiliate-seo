# Content Calendar Skill

## 概要
効率的な記事作成スケジュールを立て、継続的なコンテンツ更新を実現する

---

## Inputs（入力）

### 必須
- `keyword_list`: キーワードリスト（/keyword-researchの出力）
- `target_period`: 計画期間（デフォルト: 6ヶ月）

### オプション
- `platform`: プラットフォーム（`blogger` / `wordpress`、デフォルト: `blogger`）
- `weekly_posts`: 週間投稿数（デフォルト: 2記事/週）
- `team_size`: チーム人数（デフォルト: 1人）

---

## Outputs（出力）

### 形式
Markdown形式のコンテンツカレンダー

### 含まれる内容
1. 全体ロードマップ（Phase別）
2. 月次カレンダー（記事タイトル、タイプ、優先度）
3. 記事タイプ別配分（収益/集客比）
4. リライトスケジュール
5. KPI追跡表

---

## このスキルを使うタイミング
- サイト構造設計完了後
- 記事作成ペースを管理したいとき
- チーム運営でタスク分担するとき

---

## 実行手順

### 1. 作成スケジュール策定

#### 月2-5万円達成のための記事投稿ペース
```
目標: 6ヶ月で30-50記事

Month 1-2: 週2-3記事 (計20記事)
Month 3-4: 週1-2記事 (計15記事)
Month 5-6: 週1記事 (計10記事) + リライト

合計: 45記事 + リライト10記事
```

### 2. 記事タイプ別配分

```markdown
## 記事構成比

| タイプ | 記事数 | 割合 | 目的 |
|--------|--------|------|------|
| 収益記事 (商標) | 5-8記事 | 15% | 直接収益化 |
| 収益記事 (ランキング) | 5-8記事 | 15% | 比較・選定 |
| 集客記事 (HowTo) | 15-20記事 | 45% | PV獲得 |
| 集客記事 (情報) | 10-15記事 | 25% | 関連情報 |

**投稿順序**:
1. 集客記事を先に作成 (サイト全体の情報量アップ)
2. 収益記事を後から追加 (内部リンク先確保後)
```

### 3. 月次カレンダー作成

```markdown
## 2026年3月 コンテンツカレンダー

| 日付 | 記事タイトル | カテゴリ/ラベル | タイプ | KW | 文字数 | ステータス | 担当 |
|------|-------------|---------------|--------|-----|--------|-----------|------|
| 3/17 | 〇〇とは?初心者向けガイド | 基礎知識 | 集客 | 〇〇とは | 3000 | 作成中 | 山田 |
| 3/20 | 〇〇の選び方5つのポイント | 選び方 | 集客 | 〇〇選び方 | 3500 | 予定 | - |
| 3/24 | 〇〇おすすめランキングTOP10 | ランキング | 収益 | 〇〇おすすめ | 5000 | 予定 | - |
| 3/27 | [商品名]の口コミ・評判は? | レビュー | 収益 | [商品名]口コミ | 4000 | 予定 | - |

- Blogger: 「カテゴリ/ラベル」列はラベル名
- WordPress: 「カテゴリ/ラベル」列はカテゴリー名
```

### 4. 優先順位付けマトリクス

```
緊急度 × 重要度で優先順位決定:

高緊急 × 高重要 (★★★ 最優先)
├ トレンドキーワード記事
├ 競合が弱い穴場キーワード
└ 高収益案件のレビュー記事

低緊急 × 高重要 (★★☆ 計画的に)
├ ロングテール集客記事
├ まとめ・ランキング記事
└ カテゴリーページ

高緊急 × 低重要 (★☆☆ 余裕あれば)
├ 季節・イベント記事
└ 競合対応記事

低緊急 × 低重要 (☆☆☆ 後回し)
└ 雑記・日記系
```

### 5. リライトスケジュール

```markdown
## リライト計画

### 3ヶ月後リライト対象
- 公開後3ヶ月経過した記事
- 順位が11-30位の記事 (あと一歩で上位)
- CTRが平均以下の記事

### リライト内容
1. 競合記事の最新情報を追加
2. 不足している見出しを補完
3. 内部リンクを追加
4. 画像・表を追加
5. タイトル・メタディスクリプション改善

### リライト頻度
- 収益記事: 3ヶ月ごと
- 集客記事: 6ヶ月ごと
```

```markdown
## タスク管理

### 記事作成の標準フロー
1. キーワードリサーチ (30分)
2. 競合分析 (1時間)
3. 構成案作成 (30分)
4. 執筆 (3-4時間)
5. 画像作成・挿入 (1時間)
6. 校正・公開 (30分)

**1記事あたり**: 6-7時間

### 週次チェックリスト
- [ ] 月曜: 今週の記事テーマ確定
- [ ] 水曜: 進捗確認・軌道修正
- [ ] 金曜: 記事公開・SNS告知
- [ ] 日曜: 週次レポート作成

## KPI追跡

### 月次目標
| 指標 | 目標値 | 実績 | 達成率 |
|------|--------|------|--------|
| 新規記事数 | 10記事 | - | -% |
| 総記事数 | 30記事 | - | -% |
| 月間PV | 2,000 | - | -% |
| 検索順位TOP10 | 5KW | - | -% |
| 収益 | ¥5,000 | - | -% |

### 記事ごとのKPI
- 公開1週間後: インデックス確認
- 公開1ヶ月後: 順位チェック (目標: 30位以内)
- 公開3ヶ月後: 順位チェック (目標: 10位以内)
```

## ツール活用

- **Notion**: 記事データベース管理
