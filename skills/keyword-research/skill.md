# Keyword Research Skill

## 概要
SEOで上位表示を狙える収益性の高いキーワードを発掘し、コンテンツ戦略の基盤を作る

---

## Inputs（入力）

### 必須
- `site_theme`: サイトテーマ・ジャンル（例: 30代女性向けスキンケア）

### オプション
- `target_keyword_count`: 目標キーワード数（デフォルト: 30-50個）
- `focus_type`: 焦点タイプ（`revenue` = 収益重視 / `traffic` = 集客重視 / デフォルト: `balanced`）

---

## Outputs（出力）

### 形式
Markdown形式のキーワードリサーチ結果

### 含まれる内容
1. サマリー（総キーワード数、想定PV）
2. 収益記事キーワードリスト（優先度付き）
3. 集客記事キーワードリスト（優先度付き）
4. ロングテールキーワード候補
5. 競合SERP分析（主要キーワード）
6. キーワードマッピング（サイト構造との対応）


## このスキルを使うタイミング

### ✅ 使うべきとき
- サイトテーマが決まった直後（/affiliate-planning 完了後）
- 記事を書く前のキーワード調査（30-50個リストアップ）
- 既存記事のリライト対象を選定するとき
- 新しいカテゴリーを追加する際のキーワード洗い出し
- 競合が弱い穴場キーワードを見つけたいとき

### ❌ 使わない方がいいとき
- サイトテーマが決まっていない段階（/affiliate-planning を先に実行）
- すでにキーワードリストがあり、記事作成に入る段階
- 個別記事の構成を考える段階（/competitor-analysis へ）

### 📍 ワークフロー上の位置
```
/affiliate-planning → このスキル → /site-architecture → /content-calendar
```

## 実行手順

### 1. キーワードカテゴリー分類

#### 🎯 収益記事キーワード (5-10個)
購買意欲が高く、直接収益につながるキーワード
```
例:
- [商品名] 口コミ
- [商品名] 最安値
- [カテゴリ] おすすめ
- [商品A] vs [商品B]
- [カテゴリ] ランキング
```

#### 📈 集客記事キーワード (20-40個)
検索ボリュームが多く、間接的に収益記事へ誘導するキーワード
```
例:
- [カテゴリ] とは
- [カテゴリ] 始め方
- [カテゴリ] 選び方
- [問題] 解決方法
```

### 2. キーワード評価基準

各キーワードを以下の指標で評価:

| 指標 | 理想値 | 確認方法 |
|------|--------|----------|
| 検索ボリューム | 100-10,000/月 | Ubersuggest, ラッコキーワード |
| 競合性 | 低-中 | 上位サイトのDR確認 |
| CPC | ¥50以上 | Google広告キーワードプランナー |
| SERP分析 | 個人サイト存在 | Google検索結果確認 |

### 3. キーワードマッピング

```
サイト構造とキーワードの対応付け:

TOPページ
└── [メインキーワード]

カテゴリーページ
├── カテゴリA: [カテゴリキーワード1]
├── カテゴリB: [カテゴリキーワード2]
└── カテゴリC: [カテゴリキーワード3]

個別記事
├── 収益記事1: [商標KW]
├── 収益記事2: [比較KW]
├── 集客記事1: [HowTo KW]
└── 集客記事2: [情報系KW]
```

### 4. 検索意図の分析

各キーワードの検索意図を4分類:
- **Know**: 情報を知りたい (例: 〇〇とは)
- **Do**: 何かをしたい (例: 〇〇やり方)
- **Buy**: 購入したい (例: 〇〇おすすめ)
- **Go**: 特定サイトに行きたい (例: 〇〇公式)

### 5. 優先順位付け

以下のスコアリングで記事作成の優先順位を決定:

```
優先度スコア = (検索ボリューム ÷ 100) + (収益性 × 3) - (競合性 × 2)

収益性: 高=3, 中=2, 低=1
競合性: 高=3, 中=2, 低=1
```
