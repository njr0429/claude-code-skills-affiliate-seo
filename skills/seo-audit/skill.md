# SEO Audit Skill

## 概要
サイト全体のSEO状態を診断し、検索順位向上のための改善点を特定する

---

## Inputs（入力）

### 必須
- `site_url`: サイトURL

### オプション
- `audit_scope`: 監査範囲（`full` = 全体 / `technical` = テクニカルSEOのみ / `content` = コンテンツのみ / デフォルト: `full`）
- `focus_keywords`: 重点チェックキーワード（リスト）

---

## Outputs（出力）

### 形式
Markdown形式のSEO監査レポート

### 含まれる内容
1. 総合スコア（100点満点）
2. 重大な問題（優先度★★★）
3. 改善推奨（優先度★★☆）
4. パフォーマンス分析（順位、PV、収益）
5. アクションプラン（優先度別）
6. 次回監査日

---

## このスキルを使うタイミング

### ✅ 使うべきとき
- **サイト公開から3ヶ月後（初回監査）**
- **月1回の定期チェック（習慣化推奨）**
- 順位が伸び悩んでいるとき（11-30位で停滞）
- 大幅なアップデート前（Googleアルゴリズム変更後など）
- アクセスが急減したとき（問題の早期発見）
- WordPress移行前（現状把握）

### ❌ 使わない方がいいとき
- サイト公開から1ヶ月未満（データが少なすぎる）
- 記事が10本未満の段階（まずは記事を増やす）
- 監査より記事作成を優先すべき段階

### 📍 ワークフロー上の位置
```
（定期実施：月1回） → このスキル → 改善施策実施 → /conversion-optimizer
```

---

## 実行手順

### 1. テクニカルSEO監査

#### サイト速度チェック
```bash
# PageSpeed Insights で確認
- モバイルスコア: 目標90以上
- デスクトップスコア: 目標95以上
- FCP (First Contentful Paint): 1.8秒以下
- LCP (Largest Contentful Paint): 2.5秒以下
- CLS (Cumulative Layout Shift): 0.1以下
```

**改善策**:
- 画像を WebP/AVIF に変換
- 画像遅延読み込み (lazy loading)
- CSS/JS の最小化
- CDN 導入
- キャッシュ有効化

#### モバイルフレンドリー
```
✅ レスポンシブデザイン
✅ タップ要素のサイズ (最低48px)
✅ ビューポート設定
✅ フォントサイズ (最低16px)
```

#### インデックス状態
```bash
# Google Search Console で確認
- インデックス登録済みページ: 全ページの90%以上
- エラー: 404エラー、サーバーエラーゼロ
- 除外ページ: 重複・低品質ページを特定
```

#### robots.txt & サイトマップ
```bash
# robots.txt チェック
User-agent: *
Allow: /
Sitemap: https://example.com/sitemap.xml

# XML サイトマップ
- 全ページが含まれている
- 最終更新日が正確
- Google Search Console に送信済み
```

#### HTTPS & セキュリティ
```
✅ SSL証明書有効
✅ 全ページHTTPS化
✅ 混合コンテンツなし
✅ セキュリティヘッダー設定
```

### 2. オンページSEO監査

#### 各ページのチェック項目

```markdown
## タイトルタグ
- [ ] 30-40文字
- [ ] キーワード前方配置
- [ ] ページ固有
- [ ] 魅力的なコピー

## メタディスクリプション
- [ ] 120文字以内
- [ ] キーワード含む
- [ ] CTAを含む

## H1タグ
- [ ] 1ページ1つ
- [ ] キーワード含む
- [ ] タイトルタグと一致または類似

## H2-H6タグ
- [ ] 階層構造が正しい
- [ ] キーワード・共起語含む
- [ ] 適切な数 (H2: 3-8個)

## 本文
- [ ] 2000文字以上
- [ ] キーワード出現率 1-2%
- [ ] 共起語を自然に含む
- [ ] 読みやすい段落 (3-4行)

## 画像
- [ ] alt属性設定
- [ ] ファイル名に意味
- [ ] 適切なサイズ (<200KB)
- [ ] WebP/AVIF形式

## 内部リンク
- [ ] 3-5箇所
- [ ] 関連性が高い
- [ ] アンカーテキスト最適化

## 外部リンク
- [ ] 信頼できるソース
- [ ] rel="nofollow"が適切
- [ ] リンク切れなし
```

### 3. コンテンツ品質監査

#### E-E-A-T 評価
```markdown
## Experience (経験)
- [ ] 実体験・使用レビューあり
- [ ] オリジナル写真・動画
- [ ] 具体的な数値・事例

## Expertise (専門性)
- [ ] 専門用語を正しく使用
- [ ] 深い知識が示されている
- [ ] 最新情報にアップデート

## Authoritativeness (権威性)
- [ ] 著者情報明記
- [ ] 引用・参照元が信頼できる
- [ ] 業界の権威サイトからリンク

## Trustworthiness (信頼性)
- [ ] プライバシーポリシー
- [ ] 運営者情報
- [ ] お問い合わせフォーム
- [ ] 誤情報・誇張表現なし
```

#### コンテンツの鮮度
```bash
# 古いコンテンツの特定
- 6ヶ月以上更新されていない記事
- 情報が古くなっている記事
- 競合に負けている記事

→ リライト対象にする
```

### 4. キーワード最適化監査

```markdown
## 狙ったキーワードで上位表示できているか

| 記事URL | 狙ったKW | 現在順位 | 目標順位 | 改善必要? |
|---------|---------|---------|---------|-----------|
| /article-1/ | 〇〇おすすめ | 15位 | 10位以内 | ✅ Yes |
| /article-2/ | 〇〇選び方 | 5位 | - | ❌ No |
| /article-3/ | 〇〇とは | 25位 | 10位以内 | ✅ Yes |

## 改善アクション
- 15位記事: 文字数+500、見出し追加、内部リンク強化
- 25位記事: タイトル変更、メタディスクリプション改善、画像追加
```

### 5. 被リンク監査

```bash
# Google Search Console で確認
- 総被リンク数
- 参照ドメイン数
- アンカーテキスト分布
- スパムリンクの有無

# 質の高い被リンク獲得策
- 関連サイトへ寄稿
- SNSでシェア
- プレスリリース配信
- ディレクトリ登録
```

### 6. 競合比較監査

```markdown
## 競合サイトとの比較

| 指標 | 自サイト | 競合A | 競合B | 目標 |
|------|---------|-------|-------|------|
| ドメインパワー | 10 | 25 | 30 | 20+ |
| 総記事数 | 30 | 80 | 120 | 50+ |
| 平均文字数 | 3,000 | 4,500 | 3,800 | 4,000+ |
| 被リンク数 | 5 | 50 | 80 | 20+ |
| 月間PV | 2,000 | 15,000 | 20,000 | 5,000+ |

## ギャップ分析
- 文字数が競合より少ない → 各記事+500-1000文字
- 被リンクが少ない → 外部獲得施策
- 記事数が少ない → 月10記事ペース維持
```

## ツール一覧

### 無料ツール
- **Google Search Console**: インデックス、順位、エラー確認
- **Google Analytics 4**: トラフィック、行動分析
- **PageSpeed Insights**: ページ速度
- **Lighthouse**: 総合的な品質チェック
- **Mobile-Friendly Test**: モバイル対応確認

### 有料ツール
- **Ahrefs**: 被リンク、競合分析
- **SEMrush**: キーワード順位、サイト監査
- **Screaming Frog**: サイト全体のクロール

## 成功のポイント
- ✅ 月1回の定期監査を習慣化
- ✅ 問題は優先度をつけて段階的に解決
- ✅ 改善前後を数値で比較
- ✅ 競合との差を常に意識
