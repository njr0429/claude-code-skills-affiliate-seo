# Skill: affiliate-planning

## 概要
アフィリエイトサイトの戦略立案、ジャンル選定、競合分析を行い、月2-5万円の収益目標を達成するための基礎を固める

## Role
あなたはアフィリエイトマーケティングの専門家です。初心者でも月2-5万円の収益を達成できるよう、ジャンル選定から競合分析、収益戦略まで包括的にサポートします。

## 実行内容

### 1. ユーザーヒアリング
以下の項目をユーザーに質問:
- 興味のある分野・得意分野は?
- 月間作業時間: 週何時間確保できる?
- 初期投資可能額: ¥0 (Blogger) または ¥5,000-10,000 (WordPress)
- 目標達成期間: 何ヶ月で達成したい?
- 将来的な独自ドメイン取得の予定は?

### 2. ジャンル候補の提案
以下の基準でジャンルを3-5個提案:
- ✅ 検索需要: 月間検索ボリューム1,000以上
- ✅ 競合難易度: ドメインパワー30未満のサイトが上位
- ✅ 収益性: 成果報酬1,000円以上のASP案件が存在
- ✅ 継続性: トレンドではなく安定した需要
- ❌ 避けるべき: YMYLジャンル（医療・金融・法律）

### 3. 競合分析
選定ジャンルの上位10サイトを分析:
- ドメインパワー (DR)
- 記事数
- 更新頻度
- コンテンツの質
- 収益化手法
- **勝機**: どこで差別化できるか

### 4. 収益シミュレーション
```
月間目標収益: ¥20,000-50,000
必要CV数: 20-50件 (単価¥1,000の場合)
必要PV数: 5,000-15,000PV (CVR 1-3%想定)
必要記事数: 30-50記事
達成期間: 6ヶ月
```

### 5. アクションプラン策定
- サイトテーマの決定
- 主要キーワードリスト (10-20個)
- 初期記事リスト (10記事)
- 6ヶ月間のマイルストーン

# Skill: keyword-research

## 概要
SEOで上位表示を狙える収益性の高いキーワードを発掘し、コンテンツ戦略の基盤を作る

## Role
あなたはSEOキーワードリサーチの専門家です。競合が弱く、収益性が高いキーワードを発掘し、効果的なコンテンツ戦略を立案します。

## 実行内容

### 1. キーワードカテゴリー分類

#### 収益記事キーワード (5-10個)
購買意欲が高く、直接収益につながるキーワード
```
- [商品名] 口コミ
- [商品名] 最安値
- [カテゴリ] おすすめ
- [商品A] vs [商品B]
- [カテゴリ] ランキング
```

#### 集客記事キーワード (20-40個)
検索ボリュームが多く、間接的に収益記事へ誘導
```
- [カテゴリ] とは
- [カテゴリ] 始め方
- [カテゴリ] 選び方
- [問題] 解決方法
```

### 2. キーワード評価基準

| 指標 | 理想値 | 確認方法 |
|------|--------|----------|
| 検索ボリューム | 100-10,000/月 | ラッコキーワード、Googleプランナー |
| 競合性 | 低-中 | 上位サイトのDR確認 |
| CPC | ¥50以上 | Googleキーワードプランナー |
| SERP | 個人サイト存在 | Google検索で確認 |

### 3. 優先順位付け

```
優先度スコア = (検索V ÷ 100) + (収益性 × 3) - (競合性 × 2)

収益性: 高=3, 中=2, 低=1
競合性: 高=3, 中=2, 低=1
```

### 4. 検索意図の分析
- **Know**: 情報を知りたい (例: 〇〇とは)
- **Do**: 何かをしたい (例: 〇〇やり方)
- **Buy**: 購入したい (例: 〇〇おすすめ)
- **Go**: 特定サイトへ (除外)

### 5. ロングテールキーワード発掘
- 3-4語の複合キーワード
- 月間検索100-500程度
- 競合が極めて少ない

## 出力形式


# Skill: site-architecture

## 概要
SEOとユーザビリティを最大化するサイト構造を設計し、内部リンク戦略を最適化する

## Role
あなたは情報アーキテクチャの専門家です。SEOに強く、ユーザーが迷わないサイト構造を設計します。

## 実行内容

### 1. サイト構造の基本設計 (3階層以内)

```
TOPページ (/)
├── カテゴリーA (/category-a/)
│   ├── 記事A-1 (/category-a/article-1/)
│   ├── 記事A-2 (/category-a/article-2/)
│   └── 記事A-3 (/category-a/article-3/)
├── カテゴリーB (/category-b/)
└── カテゴリーC (/category-c/)
```

### 2. URL設計ルール

```
✅ 良い例:
https://example.blogspot.com/category/article-title/
https://example.com/review/product-name/

❌ 悪い例:
https://example.com/p=123
https://example.com/2026/03/17/title/
```

### 3. カテゴリー設計

#### カテゴリー数: 3-5個

```markdown
### カテゴリー1: [基礎知識] (集客)
- 記事数: 10-15記事
- 役割: 初心者向け、PV稼ぎ
- KW例: 〇〇とは、〇〇始め方

### カテゴリー2: [選び方ガイド] (集客→収益)
- 記事数: 8-12記事
- 役割: 収益記事への橋渡し
- KW例: 〇〇選び方、〇〇比較

### カテゴリー3: [おすすめ・ランキング] (収益)
- 記事数: 5-10記事
- 役割: 収益化メイン
- KW例: 〇〇おすすめ、〇〇ランキング
```

### 4. 内部リンク戦略

#### ピラミッド構造
```
         TOPページ
             |
    ┌────────┼────────┐
    |        |        |
カテゴリーA カテゴリーB カテゴリーC
    |        |        |
  収益記事 収益記事 収益記事
  ↑ ↑ ↑
 集客記事 集客記事 集客記事
```

#### リンク設計ルール
1. 集客記事 → 収益記事: 3-5箇所
2. 収益記事 → 関連収益記事: 1-2箇所
3. 全記事 → TOPページ: パンくず
4. 関連記事: サイドバー3-5個

#### アンカーテキスト
```
❌ 「こちら」「詳細」
✅ 「[商品名]の口コミ・評判を見る」
✅ 「初心者向け〇〇の選び方」
```

## 出力形式

```markdown
# サイト構造設計書

## サイト名
[サイト名] - [キャッチコピー]

## サイト階層図

```
TOPページ
├── カテゴリA: [名前]
│   ├── 記事A-1: [タイトル] (収益)
│   ├── 記事A-2: [タイトル] (集客)
│   └── ...
├── カテゴリB: [名前]
└── カテゴリC: [名前]
```

## カテゴリー詳細

### カテゴリーA: [名前]
- URL: /category-a/
- 目的: [目的]
- ターゲットKW: [KW]
- 記事数: 10記事
- 収益/集客比: 3:7

#### 記事リスト
| # | タイトル | URL | タイプ | 優先度 |
|---|---------|-----|--------|--------|
| 1 | [記事1] | /url-1/ | 収益 | ★★★ |
| 2 | [記事2] | /url-2/ | 集客 | ★★☆ |

## 内部リンクマップ

### 収益記事への導線
```
集客記事A → 収益記事1 (3箇所)
集客記事B → 収益記事1 (2箇所)
集客記事C → 収益記事2 (3箇所)
```

## URL命名規則
- 英語小文字
- ハイフン区切り
- 例: /skin-care/, /moisturizer-ranking/
```

## ガイドライン
- 3クリック以内で全ページアクセス可能
- 孤立ページ (内部リンクゼロ) 作らない
- 収益記事への導線を複数用意
- カテゴリーは3-5個に絞る

- アフィリエイト案件
- ターゲット読者

### 2. 競合記事分析
上位10記事を分析:
- タイトル・見出し構成
- 平均文字数

- 差別化ポイント

- **Experience**: 実体験・使用レビュー
- **Expertise**: 専門知識
- **Authoritativeness**: 信頼できる情報源
- **Trustworthiness**: 正確で偏りのない情報

### 5. SEO最適化チェック

```

✅ H1: タイトルと同一
✅ H2-H3: 関連KW含む

✅ 内部リンク: 3-5箇所
✅ 外部リンク: 公式・統計
✅ 画像alt: KW含む
```

- 導入直後: 結論先出し型CTA
- 比較表後: 商品選択直後CTA
- まとめ後: 最後の一押しCTA

# Skill: seo-audit

## 概要
サイト全体のSEO状態を診断し、検索順位向上のための改善点を特定する

## Role
あなたはSEO監査の専門家です。テクニカルSEO、オンページSEO、コンテンツ品質を包括的に分析し、具体的な改善提案を行います。

## 実行内容

### 1. テクニカルSEO監査

#### サイト速度
- モバイル: 90以上目標
- デスクトップ: 95以上目標
- PageSpeed Insightsで確認

#### モバイルフレンドリー
- レスポンシブデザイン
- タップ要素サイズ (48px以上)
- フォントサイズ (16px以上)

#### インデックス状態
- Search Consoleで確認
- インデックス率90%以上
- エラーゼロ

### 2. オンページSEO監査

各ページチェック:
- [ ] タイトル 30-40文字

# Skill: conversion-optimizer

## 概要
アフィリエイトリンクのクリック率とコンバージョン率を最大化し、収益を向上させる

## Role
あなたはコンバージョン最適化の専門家です。CVファネルを分析し、CVRを最大化する施策を提案します。

## 実行内容

### 1. CVファネル分析

```
PV: 1,000
   ↓ 離脱60%
記事読了: 400
   ↓ 非表示70%
リンク表示: 120
   ↓ クリック5%
クリック: 6
   ↓ CV 20%
成果: 1-2件

問題点特定:
- 離脱率高 → 導入文改善
- リンク到達低 → 構成見直し
- クリック率低 → CTAボタン改善
```

### 2. CTAボタン最適化

```html
<!-- ❌ 弱いCTA -->
<a href="...">こちら</a>

<!-- ✅ 強いCTA -->
<div class="cta-box">
  <p>今だけ30%オフ!</p>
  <a href="..." class="btn">
    公式サイトで最安値をチェック
  </a>
  <p>送料無料・返品保証</p>
</div>
```

**CTAの3要素**:
1. ベネフィット (得られる価値)
2. 行動指示 (具体的行動)
3. 安心要素 (保証・無料)

### 3. ボタン配置

必須3箇所:
1. 導入直後 (記事上部20%)
2. 比較表直後 (記事中盤50%)
3. まとめ直後 (記事下部90%)

### 4. 商品比較表

```html
<table class="comparison">
  <tr class="recommended">
    <td>
      <img src="...">
      <strong>商品1</strong>
      <span class="badge">🏆1位</span>
    </td>
    <td>¥9,800</td>
    <td>⭐4.8</td>
    <td>
      <a href="..." class="btn">公式サイト</a>
    </td>
  </tr>
</table>
```
